# DESIGN.MD Gallery Standard

This document is the source of truth for adding high-quality websites to the gallery. Follow it even when no prior conversation context is available.

## Batch Size

- Preferred batch: 12 websites.
- Maximum quality batch: 24 websites.
- Avoid adding 50+ sites in one pass unless explicitly asked. Large batches usually reduce curation quality.

## Selection Bar

Only add sites that are strong design references and useful for generating implementation guidance.

A good gallery site should have at least 3 of these:

- Recognizable product, brand, or interaction pattern.
- Distinct visual system, not generic template styling.
- Clear typography, spacing, navigation, and component patterns.
- Useful screenshots from the public homepage or product page.
- Transferable UI lessons for builders.
- Stable public URL that is likely to stay online.

Avoid:

- Thin landing pages with no component depth.
- Sites whose quality depends only on illustration or photography.
- Broken, paywalled, private, or login-only pages.
- Duplicate style references unless they represent different product categories.
- Sites with unclear ownership, malicious content, or unstable URLs.

## Recommended Categories

Keep each batch balanced across categories:

- AI / Devtools: OpenAI, Anthropic, Cursor, Perplexity, Vercel, Linear.
- SaaS / Finance: Stripe, Ramp, Mercury, Brex, Retool, Attio.
- Design / Creative: Figma, Framer, Webflow, Read.cv, Cosmos, A24.
- Commerce / Marketplace: Shopify, Airbnb, Graza, Arc'teryx, Teenage Engineering.
- Editorial / Brand: Apple, Nike, Nothing, Arc, The Browser Company.

## Required Fields

Every gallery item must include:

- `id`: stable slug, prefixed consistently with the existing data style.
- `name`: concise brand/product name.
- `description`: one sentence explaining why the site is a strong reference.
- `url`: display hostname without protocol.
- `href`: canonical public URL with protocol.
- `pageTypes`: 3-4 accurate page types.
- `uxPatterns`: 4-6 reusable patterns.
- `uiElements`: 5-7 concrete interface elements.
- `colors`: 4-6 named colors with hex values.
- `markdown`: complete DESIGN.md generated by the local gallery template or better.

## Metadata Quality

Descriptions should be specific, not generic.

Good:

> Finance platform dashboards, approval flows, expense workflows, and enterprise conversion pages.

Bad:

> A nice modern website with good design.

Color names must be useful:

- Good: `Stripe Purple`, `Canvas`, `Muted Slate`, `Ramp Green`.
- Bad: `Blue`, `Color 1`, `Main`, unless the source brand actually uses that name.

## DESIGN.md Quality Bar

Each generated DESIGN.md must be implementation-oriented. It should include:

- Visual theme and atmosphere.
- Color roles with exact values.
- Typography rules with sizes, weights, line heights.
- Button rules.
- Card/container rules.
- Layout principles.
- Depth/elevation.
- Do and don't guidance.
- Responsive behavior.
- Agent prompt guide.

Do not add gallery entries with empty or shallow markdown.

## Gallery Detail Compatibility

The gallery detail view depends on each item having enough data for:

- Left side style reference.
- Color Palette section.
- Typography section.
- Spacing & Shape section.
- Guidelines section.
- Right side exports:
  - DESIGN.md
  - Tailwind v4
  - CSS Variables
  - Design Tokens
  - Compact / Extended variants

Before finishing, click at least one new gallery item and confirm those surfaces render.

## Motion Standard

Motion should feel premium and controlled:

- Use scroll-linked GSAP for section entrance, card reveal, and image parallax.
- Use Framer Motion for modal/detail entry, tab switching, hover, and content transitions.
- Prefer blur + translate + opacity + slight scale.
- Avoid chaotic rotation, bouncing, excessive gradients, or decorative motion that does not clarify structure.
- Interactive motion should make state changes legible.

## Verification Checklist

Run these checks before committing:

- `pnpm exec tsc --noEmit`
- `pnpm build`
- Browser check homepage gallery count.
- Browser click one gallery item.
- Confirm DESIGN.md panel scrolls.
- Confirm all export tabs show real content.
- Confirm Compact and Extended change content.
- Confirm Copy and download target current tab content.

Known current limitation:

- `pnpm lint` may fail if `eslint` is not installed in the workspace. Report that explicitly instead of hiding it.
